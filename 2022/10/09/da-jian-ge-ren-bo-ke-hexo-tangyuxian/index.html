# 搭建个人博客 _Hexo_tangyuxian

# 1. 博客的搭建

# 2. 搭建中遇到的问题

# 2.1 ssh github.com 失败

解决办法：

windows 在 C:\WINDOWS\system32\drivers\etc 目录下修改 host 文件，在最后一行添加 github 的 ip 地址

问题分析过程：

  1. 通过指令查看在建立 ssh 连接的过程中发生了什么
> ssh -vT git@github.com

连接 github.com 的地址居然是 ::1 和 127.0.0.1 。前者是 IPV6 的 localhost 地址，后 -
者是 IPV4 的 localhost 地址。

  1. 查找 github.com 的 ip 地址可以使用 https://www.ipaddress.com/ 来查询，也可以使用 nslookup 命令
> nslookup github.com 8.8.8.8

nslookup 是域名解析工具， 8.8.8.8 是 Google 的 DNS 服务器地址。直接使用

就会使用本机已经设置好的 DNS 服务器进行域名解析， ipconfig /all 可以查看本机 DNS 服务器地址。

这个问题其实就是 DNS 解析被污染了，有 2 种可能：

  • DNS 解析被运营商劫持了
  • 使用了科学上网工具

# 2.2 hexo 报错 error：spawn failed

2.1 解决办法

2.1.2 删除.deploy_git 文件夹；

git config --global http.sslVerify "false"

2. 然后，依次执行：

git config --global core.autocrlf false
hexo clean
hexo g
hexo d

2.1.2 原因

问题大多是因为 git 进行 push 或者 hexo d 的时候改变了一些.deploy_git 文件下的内容。

# 2.3 上传资源之后要过两三分钟才会显示

# 2.4 本地访问失败

输入命令 hexo s

但是访问 localhost:4000 的时候显示不出来，并且 bash 窗口中还在报错

解决办法:

hexo clean
hexo g
hexo s

在 hexo g 的过程中可能会报黄色错误，再重重头开始执行一遍命令就可以

# 2.5 在网页端看代码窗口不正常

原因是我将 theme 下的一个配置文件删掉导致不成功。文件名为.gitkeep

# 3. 写文章

# 3.1 创建文章及发布

常见命令：

hexo new "博客名称"	 #创建一篇新博客
hexo clean	#清理缓存
hexo g		#生成静态文件(generate)
hexo d		#上传博客(deploy）
hexo s		#运行服务器，网页打开localhost:4000即可预

1. 创建新文件 / 更改文件内容 后上传文件

hexo new "我的新博客"
hexo g
hexo d

2. 删除文件 后上传文件

hexo clean
hexo g
hexo d

# 3.2 如何添加图片

  1. 首先将图片放在 images 文件夹中 (自己创建，路径与_posts 同一层)

  2. 然后将 images 后的图片名称改为自己的图片名称 (前面的为自己博客的地址)

    https://uvucunicorn.github.io/images/839RMEQ3DGT8EM6W86.png

# 4. 更换主题及配置画板娘

# 4.1 更换主题

  1. 浏览主题
  2. 克隆 github 仓库地址
  3. 更改_config.yaml 主题配置信息

# 4.1.1 地址

https://hexo.io/themes/

(也可以选择进入 hexo 官网，拉到最底下，找到浏览主题)

点击图片可以预览效果

点击标题名称可以进入仓库地址

# 4.1.2 克隆 github 仓库地址

选择好要更改的主题，进入 github 仓库复制仓库地址

克隆地址 (这里以 tangyuxian 为例)

git clone https://github.com/tangyuxian/hexo-theme-tangyuxian.git themes/tangyuxian

将中间仓库地址换为你喜欢的仓库地址，最后面的 tangyuxian 更换成你喜欢主题的名称

# 4.1.3 更改_config.yaml 主题配置信息

找到 theme, 将后面的主题换为上一步中你更改的主题名称 (也就是克隆的文件夹名称)

# 4.2 添加画板娘

4.2.1 安装插件

也可参阅教程和画板娘礼包

安装插件

npm install --save hexo-helper-live2d

在_config.yaml 文件中添加如下配置信息

例子:

live2d:
  enable: true
  scriptFrom: local
  pluginRootPath: live2dw/
  pluginJsPath: lib/
  pluginModelPath: assets/
  tagMode: false
  log: false
  model:
    use: live2d-widget-model-wanko
  display:
    position: right
    width: 150
    height: 300
  mobile:
    show: true
  react:
    opacity: 0.7

详细版:

# Live2D
## https://github.com/EYHN/hexo-helper-live2d
live2d:
  enable: true
  # enable: false
  pluginRootPath: live2dw/ # Root path of plugin to be on the site (Relative)
  pluginJsPath: lib/ # JavaScript path related to plugin's root (Relative)
  pluginModelPath: assets/ # Relative model path related to plugin's root (Relative)
  scriptFrom: local # Default
  # scriptFrom: jsdelivr # jsdelivr CDN
  # scriptFrom: unpkg # unpkg CDN
  # scriptFrom: https://cdn.jsdelivr.net/npm/live2d-widget@3.x/lib/L2Dwidget.min.js # Your custom url
  tagMode: false # Whether only to replace live2d tag instead of inject to all pages
  log: false # Whether to show logs in console
  model:
    use: live2d-widget-model-wanko # npm-module package name
    # use: wanko # folder name in (hexo base dir)/live2d_models/
    # use: ./wives/wanko # folder path relative to hexo base dir
    # use: https://cdn.jsdelivr.net/npm/live2d-widget-model-wanko@1.0.5/assets/wanko.model.json # Your custom url

普通版:

# Live2D
## https://github.com/xiazeyu/live2d-widget.js
## https://l2dwidget.js.org/docs/class/src/index.js~L2Dwidget.html#instance-method-init
live2d:
  model:
    scale: 1
    hHeadPos: 0.5
    vHeadPos: 0.618
  display:
    superSample: 2
    width: 150
    height: 300
    position: right
    hOffset: 0
    vOffset: -20
  mobile:
    show: true
    scale: 0.5
  react:
    opacityDefault: 0.7
    opacityOnHover: 0.2

4.2.2 下载画板娘以及配置

在博客根目录下创建目录live2d_models, 在目录下执行下面的命令

git clone https://github.com/fguby/live2D.git

更改配置文件_config.yaml 中刚刚添加的信息中 model.use 后添加你想要使用的模型路径 (相对路径和绝对路径都可以，初学建议绝对路径)

加下来依次执行下列指令

hexo g
hexo d
hexo s

如果遇到报错就清理一下

hexo clean
