3.问题

1.ssh github.com 失败

解决办法：

windows在C:\WINDOWS\system32\drivers\etc目录下修改host文件，在最后一行添加github的ip地址

问题分析：
  1. 通过指令查看在建立ssh连接的过程中发生了什么
> ssh -vT git@github.com

连接github.com的地址居然是::1和127.0.0.1。前者是IPV6的localhost地址，后-
者是IPV4的localhost地址。

  1. 查找github.com的ip地址可以使用https://www.ipaddress.com/来查询，也可以使用nslookup命令
> nslookup github.com 8.8.8.8

nslookup是域名解析工具，8.8.8.8是Google的DNS服务器地址。直接使用

就会使用本机已经设置好的DNS服务器进行域名解析，ipconfig /all可以查看本机DNS服务器地址。

这个问题其实就是DNS解析被污染了，有2种可能：

  • DNS解析被运营商劫持了
  • 使用了科学上网工具

2.hexo报错error：spawn failed

1.解决办法
删除.deploy_git文件夹;
输入git config --global core.autocrlf false
然后，依次执行：
hexo clean
hexo g
hexo d
2.原因
问题大多是因为git 进行push或者hexo d的时候改变了一些.deploy_git文件下的内容。

4.注意事项：

  • 上传资源之后要过两三分钟才会显示

5.创建博客及发布

常见命令：

hexo new "博客名称"	 #创建一篇新博客
hexo clean	#清理缓存
hexo g		#生成静态文件(generate)
hexo d		#上传博客(deploy）
hexo s		#运行服务器，网页打开localhost:4000即可预

1.创建新文件/更改文件内容 后上传文件

hexo new "我的新博客"
hexo g
hexo d

2.删除文件 后上传文件

hexo clean
hexo g
hexo d
